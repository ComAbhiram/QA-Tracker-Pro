Team Tracker - Advanced QA & Project Management System

Thank you for purchasing Team Tracker! This documentation will guide you through setting up and customizing the application.

1. System Requirements

  • Node.js: Version 18.x or higher
  • NPM/Yarn/PNPM: Recommended `npm`
  • Database: Supabase (PostgreSQL)

2. Installation & Setup

Step 2.1: Extract the Files

Extract the downloaded ZIP file and navigate into the source code folder.

Step 2.2: Install Dependencies

Open your terminal in the root folder and run:

npm install

Step 2.3: Environment Variables

Rename .env.example to .env.local. Open it and fill in your details:

SMTP_USER=your_smtp_email
SMTP_PASS=your_smtp_password
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key

3. Database Configuration

You need a Supabase project to run this application. Follow these steps:

  1. Create a new project at Supabase.com.
  2. Go to the SQL Editor in your Supabase dashboard.
  3. Copy the contents of database/schema.sql provided in this package.
  4. Paste it into the SQL Editor and click Run. This will set up all required tables, triggers, and Row Level Security (RLS) policies.
Important: After running the SQL script, make sure you configure your Supabase Authentication settings to enable Email/Password login.

4. Running the Application

Development Mode

To run the app locally for development, use:

npm run dev

The app will be available at http://localhost:3000.

Production Build

To build for production (e.g., when deploying to Vercel, VPS, or cPanel Node):

npm run build
npm run start

5. Deployment

The easiest way to deploy this Next.js application is through Vercel.

  1. Push your code to a GitHub/GitLab repository.
  2. Connect Vercel to your repository.
  3. Ensure you add the Environment Variables in the Vercel project settings before clicking Deploy.
